## 数据流

### 创建/打开笔记流程

```
用户输入 noteName + password
         ↓
哈希 → id = SHA-256(noteName)
         ↓
查询 Supabase → 笔记存在？
         ↓         ↓
        是        否
        ↓         ↓
    验证密码   创建新笔记
        ↓         ↓
    解密内容    空内容
        ↓         ↓
     显示      编辑
```

### 保存笔记流程

```
用户编辑 textarea
         ↓
触发 watch → 防抖 1 秒
         ↓
加密内容 + 认证字符串 → AES-256-GCM
         ↓
Base64 编码
         ↓
UPSERT 到 Supabase
         ↓
更新 UI 状态
```

## 加密层设计

### 消息格式

```
┌─────────────────────────────────────┐
│  加密后的数据格式                   │
├─────────────────────────────────────┤
│ [IV (12 bytes)]                     │  随机初始化向量
├─────────────────────────────────────┤
│ [Encrypted Data (variable length)]  │  AES-256-GCM 加密数据
├─────────────────────────────────────┤
│ [Auth Tag (16 bytes)]               │  GCM 认证标签（自动）
├─────────────────────────────────────┤
│          ↓ Base64 编码 ↓            │
│   [Base64 String]                   │  最终存储格式
└─────────────────────────────────────┘
```

### 加密时间表

```
时间 ← 用户输入
  ↓
  ┌─ deriveKey(noteName, password)
  │    SHA-256(noteName + password) → 32 字节密钥
  ↓
  ┌─ 生成随机 IV (12 字节)
  ↓
  ┌─ 加密数据
  │    crypto.subtle.encrypt(
  │      { name: "AES-GCM", iv },
  │      key,
  │      dataBuffer
  │    )
  ↓
  ┌─ 获取加密结果（包含认证标签）
  ↓
  ┌─ 合并 IV + 加密数据
  ↓
  ┌─ Base64 编码
  ↓
最终加密字符串 → 发送到 Supabase
```

## 密钥派生算法

```
Input:
  - noteName: string (笔记名称)
  - password: string (访问密码)

Process:
  combined = noteName + password
  ↓
  combined_bytes = UTF-8(combined)
  ↓
  key_hash = SHA-256(combined_bytes)  // 32 字节
  ↓
  key = ImportKey(
    key_hash,
    "AES-GCM",
    { extractable: false, usages: ["encrypt", "decrypt"] }
  )

Output:
  - key: CryptoKey (AES-256-GCM)
```

## 认证机制

```
注册密码时:
  password_auth = "VAULTNOTE_AUTH_2026"
  encrypted_auth = AES-GCM-Encrypt(key, password_auth)
  存储 encrypted_auth

验证密码时:
  尝试解密 encrypted_auth
  if decrypted == "VAULTNOTE_AUTH_2026"
    → 密码正确
  else
    → 密码错误

作用:
  防止用户用错误密码无意义地"解密"数据
```

## 并发控制

```
用户快速输入 → 多次触发 watch
  ↓
防抖处理（1 秒延迟）
  ↓
取消之前的保存计时器
  ↓
设置新的保存计时器
  ↓
只有最后一次编辑会被保存
```

## 错误处理

```
┌─ 网络错误
│  ├─ Supabase 连接失败 → 显示重试提示
│  └─ API 超时 → 显示超时提示

├─ 密码错误
│  └─ 验证失败 → 通用错误提示

├─ 加密错误
│  ├─ 加密失败 → 重试
│  └─ 解密失败 → 显示错误

└─ 输入验证
   ├─ 笔记名为空 → 提示输入
   └─ 密码太短 → 提示至少 6 位
```

## 安全边界

```
┌─────────────────────────────────────┐
│        用户浏览器（信任）           │
│                                     │
│  ┌─────────────────────────────┐   │
│  │  Web Crypto API             │   │
│  │  - 加密发生于此              │   │
│  │  - 密钥从不离开浏览器        │   │
│  └─────────────────────────────┘   │
│                                     │
└─────────────────────────────────────┘
             ↓ HTTPS  (加密传输)
┌─────────────────────────────────────┐
│    Supabase 服务器（不信任）        │
│                                     │
│  ┌─────────────────────────────┐   │
│  │  存储加密数据               │   │
│  │  - 无法解密                  │   │
│  │  - 无法访问密钥              │   │
│  │  - 只能看到加密字节          │   │
│  └─────────────────────────────┘   │
│                                     │
└─────────────────────────────────────┘
```

## 数据库查询优化

```
查询现有笔记:
  SELECT * FROM notes WHERE id = $1
  ↓
  使用 id（PRIMARY KEY）索引
  ↓
  O(1) 时间复杂度

更新笔记:
  UPSERT notes 
  ON CONFLICT (id) DO UPDATE ...
  ↓
  避免先查询再更新
  ↓
  原子性更新
```

## 扩展性考虑

```
当前设计支持:
  ✅ 百万级笔记
  ✅ TB 级数据
  ✅ 全球部署
  ✅ 高并发（Supabase 自动扩展）

限制因素:
  - 单笔记大小（PostgreSQL 限制）
  - 单次传输大小（网络限制）
  - 加密 CPU（客户端浏览器）
```

## 性能优化

```
前端优化:
  ✅ 防抖保存（减少网络请求）
  ✅ 增量加密（每次保存整个笔记）
  ✅ 延迟加载（首屏快速）

后端优化:
  ✅ 数据库索引（快速查询）
  ✅ RLS 预编译（快速权限检查）
  ✅ UPSERT 原子性（避免并发问题）

传输优化:
  ✅ HTTPS 压缩
  ✅ Base64 编码（跨平台兼容）
  ✅ 批量操作（未来可优化）
```

## 备份和恢复

```
自动备份:
  Supabase 每日自动备份
  ↓
  7 日保留期（免费）
  ↓
  手动备份可保留更久

恢复流程:
  1. Supabase 控制台 → 恢复备份
  2. 客户端代码无需修改
  3. 继续使用相同的 noteName + password
```

## 合规性

```
数据隐私:
  ✅ GDPR 兼容（用户控制数据）
  ✅ E2E 加密（即使 Supabase 也看不到）
  ✅ 无数据跟踪
  ✅ 无第三方分析（可选添加）

安全标准:
  ✅ AES-256（美国国防标准）
  ✅ SHA-256（NIST 推荐）
  ✅ HTTPS（TLS 1.2+）
  ✅ 无弱密码存储
```
